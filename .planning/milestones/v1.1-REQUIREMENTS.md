# Requirements: Mac OS Swiss Knife

**Defined:** 2026-04-13
**Core Value:** From the menu bar, I can reliably wake the devices I care about and trust the app's status without editing code or fighting the UI.

## v1 Requirements

Requirements for the v1.1 hardening milestone. This milestone does not add major end-user features; it makes the shipped v1.0 behavior easier to trust, verify, and maintain.

### Documentation Integrity

- [x] **DOC-01**: Maintainer can read current planning docs without seeing removed v1.0 scope described as shipped behavior
- [x] **DOC-02**: Maintainer can read phase verification files and trust that any reported gaps still reflect the current codebase
- [x] **DOC-03**: Maintainer can distinguish documentation debt from product blockers in current milestone audit and project context files

### Automation Strategy

- [x] **AUTO-01**: Maintainer can identify exactly which wake and management entry flows are covered by automated regression tests
- [x] **AUTO-02**: Maintainer can see one explicit verification strategy for real menu-bar entry paths, whether via automation or documented non-automation coverage
- [x] **AUTO-03**: Maintainer can run a stable regression slice for polished wake and management surfaces without ambiguous assumptions about tray-click coverage

### Validation Quality

- [x] **VAL-01**: Maintainer can open Phase 01-04 validation files and see wave-0 completion state that matches actual coverage status
- [x] **VAL-02**: Maintainer can map each Phase 01-04 validation file to concrete automated or manual verification instead of unresolved placeholder work
- [x] **VAL-03**: Maintainer can audit remaining validation debt for Phase 01-04 without re-discovering missing ownership or missing test seams

## v2 Requirements

Deferred until after the hardening milestone is complete.

### Convenience

- **CONV-04**: User can access a short recent-devices list for faster repeat wake actions

### Distribution

- **DIST-01**: App can move toward packaging hardening such as signing or notarization once the maintenance baseline is stable

## Out of Scope

Explicitly excluded from this milestone to keep v1.1 focused on hardening the shipped baseline.

| Feature | Reason |
|---------|--------|
| New Wake-on-LAN convenience features beyond hardening | Hardening should finish before adding fresh scope |
| `MenuBarExtra` migration | Too large for a debt-focused follow-up milestone |
| Cloud sync, accounts, or device sharing | Still misaligned with the local personal-use scope |
| LAN discovery or advanced networking workflows | Not required to close the current maintenance debt |
| Public release packaging polish | Valuable later, but not the highest leverage follow-up to v1.0 |

# Roadmap: Tools Cat

## Overview

This milestone returned to one narrow product-facing gap in the already-shipped saved-device flow. The device library already had native list presentation and stable CRUD truth, but the add/edit form still surfaced validation too early while the user was typing, and a first-use empty library still started completely blank even though one stable NAS target was known. The shipped goal was to improve form timing and first-use seeding without reopening the wake/menu contract.

## Phase Details

### Phase 19: Deferred Device Form Validation
**Goal**: The `设备库` add/edit form feels native and non-noisy by revealing validation only after blur or explicit submit, while invalid drafts still cannot be saved.
**Depends on**: Phase 18
**Requirements**: DEVS-10, DEVS-11, DEVS-12
**Plans**: 1/1 plans complete
**Success Criteria** (what must be TRUE):
  1. The required-name hint does not appear during in-progress typing and only appears after the user leaves the field or explicitly submits it.
  2. MAC validation hints do not appear during in-progress typing and only appear after the user leaves the field or explicitly submits it.
  3. Tapping save without valid inputs still reveals the correct validation feedback and refuses to persist an invalid saved device.

### Phase 20: First-Use Device Seed
**Goal**: A brand-new empty saved-device library gets one practical default NAS entry exactly once, without mutating existing personal libraries.
**Depends on**: Phase 19
**Requirements**: DEVS-13, DEVS-14
**Plans**: 1/1 plans complete
**Success Criteria** (what must be TRUE):
  1. A first-use empty library automatically contains exactly one saved device named `UGREEN NAS` with normalized MAC `6C:1F:F7:75:C7:0E`.
  2. Reloading after the first-use seed does not duplicate the default device.
  3. Existing non-empty libraries never receive the default seed implicitly.

## Milestone Summary

**Decimal Phases:**

- None.

**Key Decisions:**

- Keep validation truth in `DeviceLibrarySessionModel` and expose reveal-aware messages instead of moving validation into the view.
- Keep `saveDraft()` as the explicit invalid-submit boundary instead of disabling save only when the form is invalid.
- Seed exactly one default `UGREEN NAS` device inside `SavedDeviceRepository` only when the canonical saved-device payload key is absent.
- Preserve requirement ownership on Phases 19 and 20; Phase 21 closes evidence debt rather than shipping new runtime behavior.

**Issues Resolved:**

- Removed premature name and MAC validation noise from the saved-device add/edit flow.
- Added one practical first-use default NAS target so brand-new libraries are immediately useful.
- Restored missing Phase 19/20 verification artifacts and reran the milestone audit to a passed state.

**Issues Deferred:**

- `CONV-04`: recent-device shortcuts remain deferred to a future milestone.
- `AWAKE-12`: one-off unsaved timed keep-awake durations remain out of scope.
- `AWAKE-13`: custom labels or notes for managed durations remain out of scope.

**Technical Debt Incurred:**

- None beyond the explicit deferred future-scope items listed above.

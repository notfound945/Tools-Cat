<objective>
Change the saved-device add/edit form so validation feedback is revealed only after blur or explicit submit, while keeping save-time validation and persistence truth unchanged.

Purpose: Phase 19 exists because the current `设备库` form shows required/MAC warnings immediately from live draft text, which feels noisy and non-native. The phase should decouple validation truth from validation visibility and make `保存设备` an explicit submit path again instead of silently disabling it when invalid.
Output: a device-library form that stays quiet while typing, reveals field-level guidance on blur or submit, and still rejects invalid saves.
</objective>

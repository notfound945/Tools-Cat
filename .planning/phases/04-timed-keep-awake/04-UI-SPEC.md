## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight inline spacing inside any composed status text or icon-assisted helper row |
| sm | 8px | Compact gaps between related controls in any attached alert or auxiliary surface |
| md | 16px | Default inset or grouped-content spacing if this phase needs a compact secondary surface outside the menu |
| lg | 24px | Separation rhythm between keep-awake controls and adjacent menu sections conceptually |
| xl | 32px | Reserved for larger grouped empty/error blocks outside the root menu |
| 2xl | 48px | Reserved for future larger native sheets; not a default menu spacing override |
| 3xl | 64px | Not used directly in this phase; retained for milestone-wide consistency |

Exceptions: native `NSMenu` row height, menu left/right gutters, submenu indentation, and status-bar button hit area remain system-managed and must not be overridden. Source: default native macOS behavior plus `04-CONTEXT.md` compact-menu constraint.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 13px | 600 | 1.3 |
| Heading | 17px | 600 | 1.2 |
| Display | 22px | 600 | 1.2 |

Use only weights `400` and `600`. Root menu rows and the keep-awake status row use `Body` or `Label` only; `Heading` and `Display` are reserved for any attached native alert or later supporting surface, not the compact root menu. Default chosen to match existing Phase 1 menu density and current AppKit behavior.

## Color

Use macOS semantic system colors in implementation; the values below are reference targets for the contract, not hardcoded palette requirements.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `NSColor.windowBackgroundColor` (approx `#F5F5F5` light / `#1E1E1E` dark) | Root menu background and any attached compact native surface |
| Secondary (30%) | `NSColor.controlBackgroundColor` (approx `#EBEBEB` light / `#2C2C2E` dark) | Passive grouped surfaces, alert bodies, and secondary status containers |
| Accent (10%) | `NSColor.controlAccentColor` (default macOS blue, approx `#007AFF`) | Active keep-awake row checkmark, focused default button in any alert, and pending progress accessory only |
| Destructive | `NSColor.systemRed` (`#FF3B30`) | Destructive actions only; not expected to appear in this phase |

Accent reserved for: the single confirmed active keep-awake mode, the default-focused control in any attached alert, and a pending progress accessory if one is shown. Never use accent for countdown text, passive duration labels, failure copy, or the always-visible `关闭常亮` row. Non-destructive failure copy should use `NSColor.systemOrange`; passive helper text should use `NSColor.secondaryLabelColor`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 开始常亮 |
| Empty state heading | 未启用常亮 |
| Empty state body | 选择“无限常亮”或任一时长预设开始；要提前结束，使用“关闭常亮”。 |
| Error state | 未能更新常亮状态，当前状态未改变。请稍后重试。 |
| Destructive confirmation | `无`: `关闭常亮` 和模式替换都立即执行，不弹确认 |

Additional runtime copy rules:

- Keep-awake action titles are fixed and must stay stable: `无限常亮`、`15 分钟`、`30 分钟`、`1 小时`、`2 小时`、`关闭常亮`. Source: `04-CONTEXT.md` D-02 to D-05.
- Do not write countdown values into action titles. Countdown lives only in the dedicated status row. Source: `04-CONTEXT.md` D-08 to D-10.
- Active indefinite state copy uses `当前：无限常亮`.
- Active timed state copy uses `还剩 {formattedDuration}` with these exact formatting rules:
  - `>= 1 小时`: show at most two units, for example `还剩 1 小时 28 分钟`
  - `>= 1 分钟` and `< 1 小时`: show minutes only, for example `还剩 28 分钟`
  - `< 1 分钟`: switch to seconds, for example `还剩 42 秒`
- Pending copy lives only in the status row and must be mode-specific: `正在切换为无限常亮...`、`正在切换为 15 分钟常亮...`、`正在切换为 30 分钟常亮...`、`正在切换为 1 小时常亮...`、`正在切换为 2 小时常亮...`、`正在关闭常亮...`
- Failure copy preserves truthful Phase 1 semantics, for example `未能开启常亮，当前状态未改变` or `未能关闭常亮，当前状态未改变`.
- Timed expiry does not add a special completion banner or `已结束` copy. Once disable succeeds, the menu returns to the normal off presentation. Source: `04-CONTEXT.md` D-07 and D-10.

## Native Surface Contract

- This phase modifies only the root status-bar menu and status-bar icon. Do not add a new window, sheet, popover, notification, toolbar, or preferences surface. Source: `04-CONTEXT.md` phase boundary.
- Keep-awake controls remain in the root menu as one compact, directly visible group. Do not move them into a submenu. Source: `04-CONTEXT.md` D-01.
- The keep-awake group appears before the WOL and device-management actions so the feature remains a first-class top-level control without expanding the rest of the menu.
- Preserve native AppKit menu density and alignment. Do not add explanatory paragraphs, custom row chrome, badges, or attributed-string styling to the action rows.

## Menu Structure Contract

- Fixed root-menu order for the keep-awake section:
  1. `无限常亮`
  2. `15 分钟`
  3. `30 分钟`
  4. `1 小时`
  5. `2 小时`
  6. `关闭常亮`
  7. One disabled keep-awake status row directly below the action group
- The old single-row `保持屏幕常亮` toggle must be removed from the keep-awake section. Source: `04-CONTEXT.md` D-03.
- `关闭常亮` stays visible in all states so the user always has a direct manual stop path. Source: `04-CONTEXT.md` D-04.
- The keep-awake status row is the only non-action row introduced by this phase. Keep it disabled, single-line, and hidden when there is no active timed/indefinite state, no pending transition, and no failure message.

## Interaction Contract

- Exactly one keep-awake mode row may show the native AppKit checkmark at a time, and only for the currently confirmed active mode:
  - `无限常亮` checked when indefinite keep-awake is confirmed on
  - one timed preset row checked when that exact timed session is confirmed active
  - no keep-awake row checked when confirmed off
- During any pending enable, replacement, expiry shutdown, or manual shutdown, disable all six keep-awake action rows until the underlying power-controller completion resolves. Titles remain unchanged during the pending state.
- Selecting a new timed preset while another timed session is active immediately replaces the old session with the newly chosen duration. Selecting `无限常亮` while timed is active immediately replaces timed mode with indefinite mode. No confirmation step is allowed. Source: `04-CONTEXT.md` D-06.
- `关闭常亮` triggers the same truthful disable path for both indefinite and timed sessions. It is not destructive and does not require confirmation.
- Keep-awake state changes become visually confirmed only after the underlying assertion outcome returns `.success(...)` or `.unchanged(...)`. Failure keeps the previously confirmed checkmark and steady-state icon. Source: `04-CONTEXT.md` D-11, `04-RESEARCH.md`.
- Countdown state is derived from an absolute end time and refreshed live in the status row. It must feel live to the user, but the remaining time is informational only; expiry still goes through the same truthful disable flow before the UI returns to off. Source: `04-RESEARCH.md`.

## Status and Icon Contract

- Off state:
  - menu bar icon: `bolt.slash`
  - accessibility label / tooltip: `常亮已关闭`
  - no keep-awake row checked
  - keep-awake status row hidden unless a failure message is still being shown
- Confirmed indefinite state:
  - menu bar icon: `bolt.fill`
  - accessibility label / tooltip: `常亮已开启：无限常亮`
  - `无限常亮` checked
  - status row visible with `当前：无限常亮`
- Confirmed timed state:
  - menu bar icon: `bolt.fill`
  - accessibility label / tooltip: `常亮已开启：剩余 {formattedDuration}`
  - selected timed preset checked
  - status row visible with `还剩 {formattedDuration}`
- Pending state:
  - menu bar icon: `bolt.slash`
  - accessibility label / tooltip: `常亮状态更新中`
  - previously confirmed row stays checked until success or failure resolves
  - status row visible with the mode-specific pending copy
- Failure state:
  - menu bar icon and checkmark stay on the last confirmed state
  - status row shows one truthful failure message until the user makes another keep-awake selection

## Component Inventory

- Status-bar button icon: `bolt.fill` or `bolt.slash`
- Root-menu keep-awake action group: five start/replace rows plus one manual-off row
- Keep-awake status row: disabled single-line row for pending, active, countdown, and failure copy
- Existing downstream menu sections: WOL action, device management, quit action

## Out of Scope Guardrails

- Do not add custom duration entry, sliders, freeform text input, or a dedicated timer window in this phase.
- Do not add notifications, alerts on normal expiry, keyboard shortcuts, automation rules, favorites, or persistence beyond the timed-session behavior already required.
- Do not convert the root menu into a long explanatory list or split keep-awake into a separate settings surface.
- Do not add non-native decoration, custom typography, or a palette that overrides standard macOS menu colors.
